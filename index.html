Technical Whitepaper

Complete specification of the VelCoin Protocol

1Executive Summary

VelCoin (VLC) is a centralized Layer-1 blockchain infrastructure designed for digital commerce ecosystems, online marketplaces, and gaming reward systems. Unlike fully decentralized cryptocurrencies, VelCoin adopts a controlled architecture operated by a single node operator during its initial phase.

This approach enables:

  • High scalability and predictable transaction costs
  • Simplified integration for developers and merchants
  • Regulatory compliance and fraud prevention
  • Rapid ecosystem deployment with modular components

VelCoin aims to power a digital economy for content marketplaces, micro-transactions, and reward systems, with a transparent roadmap toward gradual decentralization through a federated validator model.

2Centralization Declaration

Important Legal Notice:

VelCoin currently operates under a centralized node architecture managed by the VelCoin Node Operator. This is explicitly disclosed and forms part of our regulatory compliance strategy.

We do not present VelCoin as a decentralized cryptocurrency, but rather as a:

Digital Commerce Ecosystem Blockchain

This positioning reduces regulatory risks, aligns with our marketplace and gaming focus, and facilitates enterprise adoption.

3.1Fixed Supply Model

  • Maximum Supply: 1,000,000,000 VLC (immutable)
  • Circulating Supply: Dynamically calculated from state.json
  • Distribution: Pre-allocated at genesis to Founder Wallet
  • Emission: No block rewards, no halving, no inflation

Supply verification: sum(all balances in state.json) = 1,000,000,000

3.3Economic Flow

Games → Wallet → Marketplace → Ecosystem

VelCoin maintains internal value flow through:

  • Spending VLC in marketplace for digital content
  • Developers paying API fees in VLC
  • Optional 2% burn fee creating deflationary pressure

4.1Network Architecture

Network Type: velcoin-mainnet (Layer-1)

Consensus: Centralized PoW Validation (not distributed mining)

Node Operator: Single authorized validator during Phase 1

Block Creation Process:

  1. Collect transactions from mempool
  2. Construct block header with previous hash
  3. Execute local PoW (SHA-256, 4 leading zeros)
  4. Validate all transactions
  5. Append to blockchain.json
  6. Update state.json balances

4.2Security Model

Validation Layers:

  • SHA-256 hash-linked blocks
  • PoW difficulty for block integrity
  • Nonce-based transaction ordering (replay protection)
  • ECDSA-style signature verification
  • API rate limiting (20 req/min default, 5 req/min mining)

Backup System: Automated GitHub repository backups for disaster recovery and version control.

4.3Wallet & Keys

  • Private Key: 64-character hexadecimal (local generation)
  • Public Key: SHA-256 hash of private key
  • Address: First 40 characters of public key hash
  • Security: Fully local, deterministic, non-custodial

7.1Asset Classification

VelCoin (VLC) is classified as a Digital Utility Asset designed for:

  • Marketplace payments and digital content purchases
  • Gaming rewards and in-game economies
  • Microtransactions within the VelCoin ecosystem
  • Developer API service fees

Explicitly NOT: Investment contract, security, or commodity asset. No ownership or equity rights conferred.
